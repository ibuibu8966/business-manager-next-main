# 用語集

このドキュメントでは、プロジェクトで使われる技術用語をわかりやすく説明します。

---

## 🔤 基本用語

### アプリケーション関連

| 用語 | 読み方 | 説明 |
|------|--------|------|
| **Next.js** | ネクストジェイエス | Webアプリを作るためのフレームワーク（土台）。Reactというライブラリをベースにしている |
| **React** | リアクト | Facebook（Meta）が作った、画面を作るためのライブラリ |
| **TypeScript** | タイプスクリプト | JavaScriptに「型」という機能を追加した言語。間違いを見つけやすくなる |
| **Supabase** | スーパーベース | データを保存するためのサービス。データベースを簡単に使える |
| **API** | エーピーアイ | アプリ同士がやり取りするための窓口 |

---

### ファイル・フォルダ関連

| 用語 | 説明 |
|------|------|
| **コンポーネント** | 再利用できる画面の部品。ボタンやフォームなど |
| **ページ** | ブラウザで表示される1つの画面 |
| **ルート** | URLのパス（例: `/tasks` がタスク管理画面） |

---

### データベース関連

| 用語 | 説明 |
|------|------|
| **テーブル** | データを保存する表。Excelのシートのようなもの |
| **レコード** | テーブル内の1行（1件のデータ） |
| **カラム** | テーブルの列（項目名） |
| **ID** | 各レコードを識別するための一意な番号 |
| **外部キー** | 他のテーブルのデータを参照するための項目 |
| **ER図** | テーブル同士の関係を表す図 |

---

### 開発関連

| 用語 | 説明 |
|------|------|
| **npm** | パッケージ（ライブラリ）を管理するツール |
| **Git** | ファイルの変更履歴を管理するツール |
| **GitHub** | Gitのプロジェクトを保存・共有するサービス |
| **デプロイ** | アプリを本番環境（インターネット上）に公開すること |
| **Vercel** | Next.jsアプリを簡単にデプロイできるサービス |

---

## 📊 ステータス用語

### タスクのステータス

| ステータス | 意味 |
|-----------|------|
| 未着手 | まだ手をつけていない |
| 進行中 | 作業している途中 |
| 完了 | 終わった |

### チケットのステータス

| ステータス | 意味 |
|-----------|------|
| 新規 | 新しく来た依頼 |
| 対応中 | 対応している途中 |
| 保留 | 何らかの理由で一時的に止めている |
| 完了 | 対応が終わった |

### 優先度

| 優先度 | 英語 | 意味 |
|--------|------|------|
| 高 | high | 急いで対応が必要 |
| 中 | medium | 通常の優先度 |
| 低 | low | 後回しでも良い |
